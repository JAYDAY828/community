제1조 (목적)

이 약관은 부평구금융인 BINANCE API의 이용과 관련하여, 이용자와 운영자 간의 권리·의무 및 책임사항을 규정함을 목적으로 합니다.

제2조 (운영 성격 및 후원)

  • 부평구금융인 BINANCE API는 후원 기반으로 운영됩니다.
  • 이용자는 후원 금액에 따라 일정 기간 이용 혜택을 받을 수 있으며, 기간은 운영 상황에 따라 변동될 수 있습니다.
  • 이용 제공 여부 및 기간은 후원 상황, 기술적 여건, 기타 운영 사정에 따라 조정될 수 있습니다.

제3조 (개인정보 및 접속 정보 수집)

부평구금융인 BINANCE API는 부정 사용 방지 및 안정적인 이용 제공을 위해 다음 정보를 수집·저장할 수 있습니다.

  • 접속 시간, IP 주소, 기기 정보, 시리얼키 사용 기록

수집된 정보는 부정 접속 적발 및 운영 목적 외에는 사용하지 않으며, 이용자가 탈퇴하는 시점까지 보관한 후 지체 없이 파기합니다.

다만, 관련 법령에 따라 일정 기간 보존이 요구되는 경우에는 해당 법령에서 정한 기간 동안 별도로 보관할 수 있습니다.

부정 사용이 적발된 경우, 이용자의 계정은 사전 경고 없이 영구 정지될 수 있습니다.

부평구금융인 BINANCE API를 이용하는 경우, 개인정보 수집에 동의한 것으로 간주됩니다.

제4조 (책임 제한)

  • 본 도구는 바이낸스(Binance) API를 기반으로 작동하며, 모든 거래·체결·시세 데이터는 바이낸스 시스템에서 처리됩니다.
  • 부평구금융인 BINANCE API는 이용자의 지갑이나 자산에 직접 접근할 권한이 없으며, 이용자가 API 키 발급을 통해 허용한 기능 범위 내에서만 작동합니다.
  • 운영자는 바이낸스의 서버 장애, API 변경, 지연, 오류 등으로 인한 거래 문제에 대해 어떠한 책임도 지지 않습니다.
  • 이용자의 매매 손익, 주문 오류, 네트워크 장애, 계정 설정 실수 등으로 발생한 손실에 대해 운영자는 책임을 지지 않습니다.
  • 부평구금융인 BINANCE API는 투자 자문이나 재정적 권고를 제공하지 않으며, 모든 투자 판단과 책임은 이용자 본인에게 있습니다.

제5조 (이용 제한)

이용자가 다음 각 호에 해당하는 경우 이용을 제한하거나 종료할 수 있습니다.

  • 타인의 계정 정보 도용
  • 시리얼키 불법 공유 또는 판매
  • 부평구금융인 BINANCE API의 역분석·복제·변조 행위
  • 법령 및 본 약관 위반

위 사유로 인해 발생한 불이익에 대해 운영자는 책임을 지지 않습니다.

제6조 (지식재산권)

  • 본 서비스는 바이낸스 API를 기반으로 제작되었으며, API 및 이를 통해 제공되는 모든 원천 데이터의 권리는 원저작자인 Binance (이하 “바이낸스”)에 귀속됩니다.
  • 단, 본 서비스를 구성하는 소프트웨어, 디자인, 로고, 사용자 인터페이스(UI), 그리고 데이터를 가공·분석하여 생성된 2차적 산출물에 대한 저작권 및 지식재산권은 운영자에게 귀속됩니다.
  • 이용자는 운영자의 사전 서면 동의 없이 다음 각 호의 행위를 하여서는 아니 되며, 이를 위반할 경우 이용 자격이 박탈될 수 있으며 관계 법령에 따른 법적 책임을 부담할 수 있습니다.
    • 본 서비스의 소스 코드 또는 프로그램을 역설계(Reverse Engineering), 디컴파일(Decompile), 복제하는 행위
    • 본 서비스를 수정, 임대, 배포, 판매하거나 이를 기반으로 2차적 저작물을 작성·제공하는 행위
    • 서비스에 포함된 운영자의 상표, 로고, 고유한 식별표지를 무단으로 사용하는 행위

제7조 (기타)

  • 본 약관에서 정하지 않은 사항은 관계 법령 및 상관례를 따릅니다.
  • 운영자는 필요한 경우 본 약관을 변경할 수 있으며, 변경된 약관은 공지 또는 안내 후 적용됩니다.
Binance API Key

목차

1. 발급 절차 2. API 권한 3. IP 제한 4. Order Toolkit 입력

1) 발급 절차

  1. 바이낸스 로그인 → 우측 상단 Account → API Management
  2. Create API 클릭 → System generated 선택
  3. 라벨(Label) 입력 (예: futures-toolkit)
  4. Security Verification 보안 인증 (이메일 / 휴대폰 또는 OTP)

2) API 권한 (API restrictions)

  • Enable Reading — 기본값 유지(✓)
  • Enable Futures — 체크(✓)
  • 그 외 항목 — 모두 해제(✗)
안내: 본 가이드는 Enable Futures 옵션만 허용하며, 발급된 API 키는 본인 외에는 누구도 접근할 수 없습니다. 운영자는 개인 자산(현물, 출금 등)에 대한 권한이 전혀 없으니 안심하셔도 됩니다.

3) IP 제한 (IP Access Restriction)

  1. Restrict access to trusted IPs only (Recommended) 선택
  2. 브라우저에서 네이버 검색창에 내 IP 입력 → 표시된 IP 주소 복사
  3. 복사한 IP를 입력 칸에 붙여넣기 → 저장(Save)
허용된 IP 이외 환경에서는 API가 동작하지 않습니다.

4) Order Toolkit에 입력

  1. 발급된 API Key와 Secret Key를 복사
  2. Order Toolkit 실행 → 상단 메뉴의 API 항목에서 키 입력/저장
주의: Secret Key는 생성 시 1회만 표시되며, 분실 시 재확인 불가입니다. 안전한 비밀관리 도구에 즉시 저장하세요.
Crypto Detectorv5

목차

1. 소개 2. 로그인 3. 필터값 & 감지 로직 4. 메인 UI & 버튼 5. 시작 기본 상태 6. 클립보드 연동 7. 로그·히스토리 8. 기타 안내 9. 문제 해결

1) 소개

Crypto Detector는 조건에 맞는 필터로 빠르게 종목을 감지하고, 변동률/거래대금/미체결(OI) 정보를 기반으로 즉시 모니터링할 수 있는 툴킷입니다.

클립보드 복사 기능을 통해 감지 종목으로 즉시 이동할 수 있습니다.

동일 기능의 트레이딩뷰 지표도 제공되며, 트레이딩뷰의 Alerts 기능을 통해 모바일 기기 등에서 실시간으로 알람을 받아볼 수 있습니다.

2) 로그인

  • 시리얼 키: 발급된 시리얼 키를 입력하여 로그인합니다. 키의 무단 공유 및 부정 사용이 적발될 경우 경고 없이 비활성화됩니다.
  • 로그인 실패: 로그인에 연속으로 실패할 경우 보안을 위해 차단될 수 있으니 주의하시기 바랍니다.
  • 차단 해제: 차단된 경우, 운영자에게 문의하여 복구를 요청해야 합니다.

3) 필터값 & 감지 로직

감지 기준

  • CHANGE %: 최근 1분 가격 등락률이 설정값 이상일 때 감지 (예: 3.0 → 1분간 3% 이상 변동 시)
  • 24H VOLUME: 24시간 거래대금이 설정값 이상일 때 감지 (예: 1,000,000 → 24h 거래대금 ≥ 1M USDT)

두 조건을 모두 충족해야 실시간 리스트에 표시됩니다.

표시 항목

  • 심볼 / 1분 등락률 / 24시간 등락률 / 1분 거래대금 / 24시간 거래대금 / 미체결(OI) / 감지 시간

4) 메인 UI & 버튼

버튼 영역

  • ▶ · ■ — 감시 시작/중지
  • 🧹 — 감지 목록 전체 초기화
  • 📌 · 📍 — 창 항상 위 토글 (📍: 항상 위 / 📌: 해제)
  • 🔔 — 알람 설정 (사운드/팝업, 볼륨, 클립보드 연동 등)
  • 🚫 — 감지 제외 심볼 관리(팝업에서 추가/삭제)
  • LOG — 1주일 로그 및 TOP 10 감지 내역 보기

리스트 영역

  • 필터 충족 종목을 실시간으로 추가·정렬
  • 감지 팝업(5초)으로 심볼·등락률·거래대금을 빠르게 확인
  • 창 위치·필터·버튼 상태 등은 자동 저장·복원

5) 시작 기본 상태

  • 감시(▶): OFF
  • 알람(🔔): UP/DOWN 알람 기본 활성화
  • 클립보드 연동: OFF (알람 설정 🔔에서 활성화 가능)
  • 항상위(📌): OFF
  • 필터 값: 기본값 표시

버튼/설정은 자동 저장되어, 재실행 시 복원됩니다.

6) 클립보드 연동 (바이낸스 URL)

감지된 종목의 정보를 클립보드로 자동 복사하는 기능입니다. 알람 설정(🔔) 창에서 활성화할 수 있습니다.

  • 활성화: 설정 창에서 'CLIPBOARD SYNC' 체크박스를 켭니다.
  • 포맷 선택:
    • Binance URL: 기본값. https://www.binance.com/en/futures/BTCUSDT 형식으로 복사됩니다.
    • Symbol Only: 심볼 이름(BTC)만 복사합니다.
    • Custom: 사용자가 {SYMBOL} 플레이스홀더를 사용하여 원하는 형식으로 URL을 직접 만들 수 있습니다.
  • 동작: 기능이 활성화되면, 종목이 감지되거나 리스트의 심볼명을 클릭할 때 설정된 포맷으로 내용이 클립보드에 복사됩니다.
  • 복사 시 프로그램 상단에 [ Copied : BTC ] 알림이 표시됩니다.

7) 로그·히스토리

  • LOG 화면에서 최근 1주 로그와 TOP 10 감지 내역을 확인할 수 있습니다.
  • Top 10 로그를 통해 모멘텀을 파악할 수 있습니다.

8) 기타 안내

  • 알람 설정(🔔): 사운드 팩, UP/DOWN 별 사운드 및 팝업 ON/OFF, 알람 볼륨을 세부적으로 제어할 수 있습니다.
  • 감지 제외 리스트(🚫): 등록된 심볼은 모든 알림·표시·사운드에서 제외됩니다.
  • 일부 기능(팝업/사운드 등)은 OS·권한 설정에 따라 제한될 수 있습니다.
  • 바이낸스 사용자라면 클립보드 연동 기능을 적극 활용해 즉시 차트/거래 화면으로 이동하세요.

9) 문제 해결

  • 리스트가 비어 있음: 필터값이 과도하게 높을 수 있습니다. 장세에 맞춰 조정하세요.
  • URL 복사 안 됨: 알람 설정(🔔)에서 'CLIPBOARD SYNC' 옵션이 활성화되어 있는지 확인하세요.
  • 팝업/사운드 미출력: 알람 설정(🔔)에서 각 항목이 켜져 있는지 확인하고, SFX 폴더의 파일 누락을 점검하세요.
Order Toolkitv15

목차

1. 소개 2. 시리얼 키 인증 3. 상단 메뉴 (레버리지·마진·심볼·API) 4. 종목 리스트 & 연동 5. 주문 설정 (ORDER·BBO·PRICE) 6. 리스크·보호 옵션 (Auto Stop Loss, Price Protection) 7. 주문/포지션 실행 8. 잔고·코스트·설정 9. SFX·테마 10. 단축키 11. 크롬 디버깅 모드 연동 12. 문제 해결

1) 소개

Order Toolkit은 바이낸스 USDⓂ 선물 거래를 빠르고 일관적으로 실행하기 위한 주문 툴입니다. 기본적으로 디버깅 모드 크롬 브라우저와 연동되며, 필요 시 Manual Mode로 단독 운용할 수 있습니다.

주요 특징

  • 그리드 자동호가 지정가 주문
    최근 고빈도봇으로 인한 호가 밀림 현상에 효과적으로 대응합니다.
    - 수수료 절약 효과
    - 호가 관통 및 밀림 상황에 최대한 대응 가능
  • 오토스탑마켓 지원
    돌파 진입뿐 아니라 급격한 가격 변동, 예기치 못한 패턴 발생 시 자동으로 스탑마켓 주문을 실행하여 다양한 상황에서 리스크를 효과적으로 관리할 수 있습니다.
  • 간편한 입력 방식
    수량 단위 입력과 달러 단위 입력을 모두 지원하여 빠르고 직관적인 주문 환경을 제공합니다.
  • 바이낸스 정식 API
    수수료나 기능 편의를 위해 안정성이 낮은 중소 거래소를 이용할 필요가 없습니다.
    글로벌 1위 거래소 바이낸스의 정식 API를 사용하여, 안전한 자산 보호와 높은 신뢰도를 기반으로 안정적인 거래 환경을 제공합니다.

2) 시리얼 키 인증

  • Serial Key Verification 다이얼로그에서 키를 입력하고 REMEMBER ME 체크 시 마지막 키가 암호화 저장됩니다.
  • 시리얼 키 부정사용 적발시 경고없이 삭제됩니다.
  • 3회 실패 시 프로그램이 종료됩니다.

3) 상단 메뉴

레버리지

드롭다운에서 1x~125x 선택. 현재 포지션 정보와 동기화되어 표시됩니다. * 배율별 맥스 포지션 한도를 확인하세요.

마진 타입

Cross / Isolated 토글 버튼. 바이낸스 계정 설정과 실시간 동기화됩니다.

심볼(SYMBOL)

예: BTCUSDT, btc, eth (대소문자 무관). 입력창 클릭 시 자동완성. 기본적으로 크롬과 종목 동기화.

API

바이낸스에서 발급받은 API 키 리스트를 관리합니다.

4) 종목 리스트 & 연동

  • Add/Remove로 현재 심볼을 리스트에 추가/삭제.
  • Auto: 크롬 디버깅 포트(기본 9222)의 바이낸스 선물 탭 URL에서 /futures/<SYMBOL>USDT 패턴을 감지하여 자동 동기화.
  • Manual: 심볼을 수동 전환.

Price Protection

Order Bookv4

목차

1. 소개 2. 메인 UI 및 기능 3. 상단 컨트롤 4. 호가창 테이블 상세 5. 크롬 디버깅 모드 연동 6. 문제 해결

1) 소개

바이낸스 웹사이트의 호가창은 스냅샷(Snapshot) 방식으로 제공되어 실시간성이 떨어집니다. 이러한 한계를 극복하고 스캘핑 거래의 결정적인 순간 포착을 위해 설계된 리얼타임(Real-time) 오더북 뷰어입니다.

주요 특징

  • 모든 호가 및 체결 물량은 실시간 USDT 가치로 자동 환산되어, 주문 규모를 매우 직관적으로 파악할 수 있습니다.
  • 3초 누적 거래대금을 통해 웹에서는 볼 수 없는 순간적인 매수/매도 힘의 세기를 시각적으로 파악할 수 있습니다.
  • 실시간으로 쌓이는 호가를 통해 주요 매물대(지지/저항)를 빠르게 확인할 수 있습니다.
  • Order Toolkit과의 연동으로, 호가창에서 확인한 가격을 클릭 한 번으로 주문창에 즉시 반영하여 신속한 거래를 지원합니다.
  • 선물(Futures)뿐만 아니라 현물(Spot) 호가를 지원하며, 강력한 모아보기(Aggregation) 필터를 통해 시장 깊이를 다각도로 분석할 수 있습니다.

2) 메인 UI 및 기능

  • 상단 컨트롤 영역: 모아보기 단위, 마켓(선물/현물), 설정 저장 방식을 제어합니다.
  • 호가창 테이블: 매도(Ask), 매수(Bid) 호가와 잔량을 실시간으로 표시합니다.
  • 체결가 영역 (중앙): 마지막 체결가와 방향(↑/↓), 3초 누적 매수/매도 대금을 표시합니다.
  • 하단 총합 영역: 현재 호가창에 표시된 매수/매도 잔량의 총합(USDT 환산)을 보여줍니다.

3) 상단 컨트롤

GLOBAL / SYMBOL 버튼

모아보기(Aggregation) 설정을 저장하는 방식을 선택합니다.

  • GLOBAL: 모든 심볼에 대해 마지막 설정을 공통으로 적용합니다.
  • SYMBOL: 각 심볼별로 모아보기 설정을 개별적으로 저장하고 복원합니다.

모아보기 (1, 10, 100, 1000)

호가를 묶어서 볼 단위를 선택합니다. 예를 들어 '10'을 선택하면 틱 사이즈의 10배 단위로 호가를 그룹화하여 보여줍니다.

FUTURES / SPOT 버튼

선물(Futures) 마켓과 현물(Spot) 마켓 간에 전환합니다. 해당 심볼이 현물 마켓에 상장되어 있을 때만 활성화됩니다.

4) 호가창 테이블 상세

  • 컬러: 매도(Ask)는 빨간색, 매수(Bid)는 녹색으로 표시됩니다.
  • 컬럼 구성: 좌측/우측 열에는 해당 호가의 주문량을 USDT 가치로 환산하여 표시하고, 중앙 열에는 실제 호가(가격)를 표시합니다.
  • 대량 주문 강조 (●): 설정된 기준(평균 거래량 대비 비율, 절대값)을 초과하는 큰 규모의 주문이 있는 호가에 ● 아이콘을 표시하여 시각적으로 강조합니다.
  • 3초 누적 거래대금: 중앙 체결가 좌우에 표시되며, 최근 3초 동안 체결된 시장가 주문의 매수/매도 대금 총합을 보여주어 순간적인 매매 강도를 파악할 수 있습니다.
  • 상호작용: 호가창의 가격(중앙 컬럼)을 클릭하면 해당 가격이 클립보드에 복사됩니다.

5) 크롬 디버깅 모드 연동(자동 심볼 감지)

Order Toolkit과 동일하게, 디버깅 모드로 실행된 크롬 브라우저의 바이낸스 종목 URL을 감지하여 자동으로 심볼을 전환합니다.

6) 문제 해결

  • 데이터가 표시되지 않음: 네트워크 연결 상태나 바이낸스 서버 상태를 확인하세요. 잠시 후에도 문제가 지속되면 프로그램을 재시작해보세요.
  • 크롬 연동이 안 됨: 크롬이 디버깅 모드로 정상 실행되었는지, 포트 번호가 9222로 맞게 설정되었는지 확인하세요.
  • 프로그램이 느려짐: 바이낸스 서버 폭주시 간헐적으로 문제가 발생할 수 있습니다. 재시작하면 대부분 해결됩니다.
  • 오류 발생 시: 프로그램 폴더에 생성되는 log_orderbook.log 파일을 확인하면 원인 파악에 도움이 될 수 있습니다.

1) 소개

Trades는 바이낸스의 실시간 체결내역 과 청산 주문 이벤트 뷰어입니다.

바이낸스 웹에서 제공되는 체결내역은 스냅샷 형태로 제공되기 때문에, 실시간 주요 물량을 확인하기에는 한계가 있습니다.

본 프로그램은 거래대금 필터를 통해 원하는 체결건만 선별적으로 확인할 수 있으며, 청산 감지 기능을 통해 단기 변곡 구간에서 발생하는 시그널을 포착할 수 있습니다.

또한 3초 누적 대금을 통해 순간적인 매수·매도 힘의 강도를 파악할 수 있습니다.

본 프로그램은 디버깅 모드 크롬 브라우저와 연동됩니다.

2) 메인 UI

가격/체결 테이블

  • 좌측 열: 가격
  • 우측 열: 체결 규모(USD) — price × qty
  • 컬러 코딩: BUY, SELL
  • 강조: 설정 임계값 이상 체결은 폰트 확대/굵게 표시

상단 정보·컨트롤

  • Title Bar: Cap : 시가총액 - 마지막 수신시각 형식
  • 심볼 라벨: 현재 스트림 심볼(예: BTCUSDT)
  • 순간 대금: 3초 누적 매수/매도 시장가 체결건 USDT 환산.

메인 버튼

  • PIN: 창을 항상 위에 고정(토글). 상태는 설정에 저장·복원됩니다.
  • LIQ: 청산 다이얼로그 열기/닫기 (자세한 설명은 아래 참조)
  • PERP: 선물/현물 모드 전환(현물 지원 심볼일 때만 Spot 활성)
  • SET: 설정 다이얼로그 열기

3) 청산(Force Order) 다이얼로그

바이낸스 forceOrder 이벤트를 실시간 표로 표시합니다. 현재 UI 심볼과 일치하는 청산만 기록하여 과거 잔여 콜백을 차단합니다.

표시 항목

  • TIME / SIDE / QTY / PRICE / VALUE
  • 가치(Value)는 K/M/B 단위 축약 표기
  • 최대 100건 유지, 최소값 필터로 실시간 정제

옵션

  • AUTO POPUP (3s): 신규 청산 발생 시 자동 팝업(3초 후 자동 숨김)
  • ALERT SOUND: 청산 알림음 재생(중복 재생 방지)
  • MIN SIZE: 이 값 이상인 청산만 표/알림에 반영

NEW 배지 & 읽지 않음 표시

오토팝업을 끄고 창이 닫혀 있는 상태에서 조건 충족 청산이 발생하면, 메인 UI의 LIQ 버튼에 NEW 깜빡임이 표시됩니다.

통계 라벨

  • 1M: 최근 1분 청산 LONG/SHORT 누적
  • 3S: 최근 3초 청산 LONG/SHORT 누적

4) 현물/선물 전환 (PERP/Spot)

  • 후원 방법과 구체적인 안내는 가입 신청이 검토·승인된 후 운영자가 연락드립니다.

후원 옵션

레퍼럴 링크를 통해 가입하고 지속적인 거래대금이 발생 시 안정적인 혜택 제공이 가능합니다.

모든 후원 혜택에는 Crypto Detector가 포함되어 있으며, 단독 이용을 원하는 경우 아래의 단독 옵션을 선택할 수 있습니다.

후원 방식은 USDT 입금 / 투네이션 / 유튜브 멤버십 가입 중 하나를 선택하여 이용할 수 있습니다.

바이낸스 레퍼럴 정책에 따라 레퍼럴 가입자의 일정 거래대금이 충족되어 커미션 요율이 상승하면 셀퍼럴 형태로 페이백 될 수 있도록 요율을 조정할 예정입니다.

  • 레퍼럴 가입 ( 거래대금 유지 조건 )
  • 7일 체험 ( 구독 및 커뮤니티 입장 조건 )
  • 30일 이용 ( 권장 후원 100 USDT )
  • 90일 이용 ( 권장 후원 260 USDT )
  • 1년 이용 ( 권장 후원 960 USDT )
  • 무제한 ( 권장 후원 3000 USDT )
  • Crypto Detector 단독 30일 이용 ( 권장 후원 30 USDT )

USDT 후원 유의사항

  • 국내외 거래소 및 지갑 이용 시 트래블룰 규정에 따라 출금·입금이 제한될 수 있습니다. 반드시 본인 계정에서 정상적인 절차로 송금해주시기 바랍니다.
  • 사용되는 네트워크(TRC20, ERC20 등)는 상황에 따라 달라질 수 있으므로 안내드린 지갑 주소와 네트워크를 반드시 확인 후 송금해주시기 바랍니다.
  • 블록체인 특성상 송금 후 거래는 취소하거나 변경할 수 없으며, 송금 이후 환불은 불가능합니다.
  • 주소 입력 오류, 네트워크 선택 오류 등으로 발생한 문제에 대해서는 운영자가 책임지지 않습니다. 송금 전 반드시 네트워크와 금액을 재확인해 주시기 바랍니다.
