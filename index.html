제1조 (목적)

이 약관은 부평구금융인 BINANCE API의 이용과 관련하여, 이용자와 운영자 간의 권리·의무 및 책임사항을 규정함을 목적으로 합니다.

제2조 (운영 성격 및 후원)

  • 부평구금융인 BINANCE API는 후원 기반으로 운영됩니다.
  • 이용자는 후원 금액에 따라 일정 기간 이용 혜택을 받을 수 있으며, 기간은 운영 상황에 따라 변동될 수 있습니다.
  • 이용 제공 여부 및 기간은 후원 상황, 기술적 여건, 기타 운영 사정에 따라 조정될 수 있습니다.

제3조 (개인정보 및 접속 정보 수집)

부평구금융인 BINANCE API는 부정 사용 방지 및 안정적인 이용 제공을 위해 다음 정보를 수집·저장할 수 있습니다.

  • 접속 시간, IP 주소, 기기 정보, 시리얼키 사용 기록

수집된 정보는 부정 접속 적발 및 운영 목적 외에는 사용하지 않으며, 보관 기간은 최대 1년입니다.

부정 사용이 적발된 경우, 이용자의 계정은 사전 경고 없이 영구 정지될 수 있습니다.

부평구금융인 BINANCE API를 이용하는 경우, 개인정보 수집에 동의한 것으로 간주됩니다.

제4조 (책임 제한)

  • 본 도구는 바이낸스(Binance) API를 기반으로 작동하며, 모든 거래·체결·시세 데이터는 바이낸스 시스템에서 처리됩니다.
  • 부평구금융인 BINANCE API는 이용자의 지갑이나 자산에 직접 접근할 권한이 없으며, 이용자가 제공한 API 키 발급을 통해 허용한 기능 범위 내에서만 작동합니다.
  • 운영자는 바이낸스의 서버 장애, API 변경, 지연, 오류 등으로 인한 거래 문제에 대해 어떠한 책임도 지지 않습니다.
  • 이용자의 매매 손익, 주문 오류, 네트워크 장애, 계정 설정 실수 등으로 발생한 손실에 대해 운영자는 책임을 지지 않습니다.
  • 부평구금융인 BINANCE API는 투자 자문이나 재정적 권고를 제공하지 않으며, 모든 투자 판단과 책임은 이용자 본인에게 있습니다.

제5조 (이용 제한)

이용자가 다음 각 호에 해당하는 경우 이용을 제한하거나 종료할 수 있습니다.

  • 타인의 계정 정보 도용
  • 시리얼키 불법 공유 또는 판매
  • 부평구금융인 BINANCE API의 역분석·복제·변조 행위
  • 법령 및 본 약관 위반

위 사유로 인해 발생한 불이익에 대해 운영자는 책임을 지지 않습니다.

제6조 (기타)

  • 본 약관에서 정하지 않은 사항은 관계 법령 및 상관례를 따릅니다.
  • 운영자는 필요한 경우 본 약관을 변경할 수 있으며, 변경된 약관은 공지 또는 안내 후 적용됩니다.
Binance API 키 생성

목차

1. 간단 생성 절차 2. API 권한 3. IP 제한 4. Order Toolkit 입력

1) 간단 생성 절차

  1. 바이낸스 로그인 → 우측 상단 Account → API Management
  2. Create API 클릭 → System generated 선택
  3. 라벨(Label) 입력 (예: futures-toolkit)
  4. Security Verification 수행 (이메일/휴대폰 또는 OTP)

2) API 권한 (API restrictions)

  • Enable Reading — 기본값 유지(✓)
  • Enable Futures — 체크(✓)
  • 그 외 항목 — 모두 해제(✗)
안내: 본 가이드는 Enable Futures 옵션만 허용하며, 발급된 API 키는 본인 외에는 누구도 접근할 수 없습니다. 운영자는 개인 자산(현물, 출금 등)에 대한 권한이 전혀 없으니 안심하셔도 됩니다.

3) IP 제한 (IP Access Restriction)

  1. Restrict access to trusted IPs only (Recommended) 선택
  2. 브라우저에서 네이버 검색창에 내 IP 입력 → 표시된 IP 주소 복사
  3. 복사한 IP를 입력 칸에 붙여넣기 → 저장(Save)
허용된 IP 이외 환경에서는 API가 동작하지 않습니다.

4) Order Toolkit에 입력

  1. 발급된 API Key와 Secret Key를 복사
  2. Order Toolkit 실행 → 상단 메뉴의 API 항목에서 키 입력/저장
주의: Secret Key는 생성 시 1회만 표시되며, 분실 시 재확인 불가입니다. 안전한 비밀관리 도구에 즉시 저장하세요.
Crypto Detectorv4.5.2

목차

1. 소개 2. 로그인 3. 필터값 & 감지 로직 4. 메인 UI & 버튼 5. 시작 기본 상태 6. 클립보드(바이낸스 URL) 7. 로그·히스토리 8. 기타 안내 9. 문제 해결

1) 소개

Crypto Detector는 조건에 맞는 필터로 빠르게 후보 종목을 추출하고, 변동률/거래대금/미체결(OI) 정보를 기반으로 즉시 모니터링할 수 있는 감지 도구입니다.

바이낸스 사용자는 클립보드 복사 기능을 통해 감지 종목으로 즉시 이동할 수 있습니다.

2) 로그인

  • 런처: 업데이트 시에만 실행하면 되며, 매번 런처로 실행해도 무방합니다.
  • 로그인키:시리얼 키 부정사용 적발시 경고없이 삭제됩니다.

3) 필터값 & 감지 로직

감지 기준

  • CHANGE %: 최근 1분 가격 등락률이 설정값 이상일 때 감지 (예: 3.0 → 1분간 3% 이상 변동 시)
  • 24H VOLUME: 24시간 거래대금이 설정값 이상일 때 감지 (예: 1,000,000 → 24h 거래대금 ≥ 1M USDT)

두 조건을 모두 충족해야 실시간 리스트에 표시됩니다.

표시 항목

  • 심볼 / 등락률 / 현재가 / 1분 거래대금 / 24시간 거래대금 / 미체결(OI) / 감지 시간

4) 메인 UI & 버튼

버튼 영역

  • ▶ · ■ — 감시 시작/중지
  • 🧹 — 감지 목록 전체 초기화
  • 📌 · 📍 — 창 항상 위 토글 (📍: 항상 위 / 📌: 해제)
  • 🔔 — 알람 설정 관리(사운드팩 커스터마이징 가능)
  • 📋 · 📑 — 바이낸스 URL 자동 복사 ON/OFF
    (활성 시 종목명 클릭으로도 주소 복사)
  • 🚫 — 감지 제외 심볼 관리(팝업에서 추가/삭제)
  • LOG — 1주일 로그 및 TOP 10 감지 내역 보기

리스트 영역

  • 필터 충족 종목을 실시간으로 추가·정렬
  • 감지 팝업(5초)으로 심볼·가격·거래대금을 빠르게 확인
  • 창 위치·필터·버튼 상태 등은 자동 저장·복원

5) 시작 기본 상태

  • 감시(▶): OFF
  • 알람(🔔): ON
  • 클립보드(📑): OFF
  • 항상위(📌): OFF
  • 필터 값: 기본값 표시

버튼/설정은 자동 저장되어, 재실행 시 복원됩니다.

6) 클립보드(바이낸스 URL)

  • ON(📋) 감지시 자동으로 클립보드에 복사되며, 리스트의 심볼을 클릭해도 해당 선물 URL이 자동 복사됩니다.
  • 예: BTC 클릭 → https://www.binance.com/en/futures/BTCUSDT
  • 복사 시 상단에 [ Copied : BTC ] 알림 표시
  • OFF(📑) 상태에서는 클릭 동작 없음

7) 로그

  • LOG 화면에서 최근 1주 로그와 TOP 10 감지 내역을 확인할 수 있습니다.
  • Top 10 로그를 통해 모멘텀을 파악할 수 있습니다.

8) 기타 안내

  • 감지 제외 리스트(🚫): 등록된 심볼은 모든 알림·표시·사운드에서 제외됩니다.
  • 일부 기능(팝업/사운드 등)은 OS·권한 설정에 따라 제한될 수 있습니다.
  • 바이낸스 사용자라면 클립보드 URL 복사를 적극 활용해 즉시 차트/거래 화면으로 이동하세요.

9) 문제 해결

  • 리스트가 비어 있음: 필터값이 과도하게 높을 수 있습니다. 장세에 맞춰 조정하세요.
  • URL 복사 안 됨: 클립보드가 📋(ON) 상태인지 확인하세요.
  • 팝업/사운드 미출력: OS 알림 권한·오디오 장치를 확인하세요.
Order Toolkitv15.0.5

목차

1. 소개 2. 시리얼 키 인증 3. 상단 메뉴 (레버리지·마진·심볼·API) 4. 종목 리스트 & 연동 5. 주문 설정 (ORDER·BBO·PRICE) 6. 리스크·보호 옵션 (Auto Stop Loss, Price Protection) 7. 주문/포지션 실행 8. 잔고·코스트·설정 9. SFX·테마 10. 단축키 11. 크롬 디버깅 모드 연동 12. 문제해결

1) 소개

Order Toolkit은 바이낸스 USDⓂ 선물 거래를 빠르고 일관적으로 실행하기 위한 주문 툴입니다. 기본적으로 크롬 브라우저와 연동되며, 필요 시 Manual Mode로 단독 운용할 수 있습니다.

권장: 실거래 전 테스트 모드에서 UI/단축키를 충분히 숙지하세요.

2) 시리얼 키 인증

  • Serial Key Verification 다이얼로그에서 키를 입력하고 REMEMBER ME 체크 시 마지막 키가 암호화 저장됩니다.
  • 시리얼 키 부정사용 적발시 경고없이 삭제됩니다.
  • 3회 실패 시 프로그램이 종료됩니다.

3) 상단 메뉴

레버리지

드롭다운에서 1x~125x 선택. 현재 포지션 정보와 동기화되어 표시됩니다. * 배율별 맥스 포지션 한도를 확인하세요.

마진 타입

Cross / Isolated 토글 버튼. 바이낸스 계정 설정과 실시간 동기화됩니다.

심볼(SYMBOL)

예: BTCUSDT, btc, eth (대소문자 무관). 입력창 클릭 시 자동완성. 기본적으로 크롬과 종목 동기화.

API

바이낸스에서 발급받은 API 키 리스트를 관리합니다.

4) 종목 리스트 & 연동

  • Add/Remove로 현재 심볼을 리스트에 추가/삭제.
  • Auto: 크롬 디버깅 포트(기본 9222)의 바이낸스 선물 탭 URL에서 /futures/<SYMBOL>USDT 패턴을 감지하여 자동 동기화.
  • Manual: 심볼을 수동 전환.
Auto 모드에서 크롬이 디버깅 모드로 실행되지 않으면 연동되지 않습니다.

5) 주문 설정

ORDER

  • Limit, Market, Stop Market 지원

BBO (가격 기준)

  • Queue n: 현재 호가창 n번째 가격 기준 지정가
  • Counterparty n: 상대 최우선 호가 n번째 기준
  • Grid x%: 현재가 ±범위에 다중 분할 주문 (스캘핑 추천 옵션)

BBO 버튼 ON 시 PRICE 입력은 자동/잠금. OFF 시 수동 입력 가능.

PRICE 빠른 지정

PRICE 라벨 좌클릭 → 매수 기준(매도1호가), 우클릭 → 매도 기준(매수1호가)로 즉시 세팅.

BTC, ETH 같은 폭발적인 거래량으로 인한 과부하 시에는 가격 지정 후 포지션 진입하는 것을 권장합니다.

SYNC

클립보드 가격연동. 트레이딩뷰 우클릭 가격 복사 및 호가 API에서의 가격 동기화.

Order Book 툴킷과도 연동됩니다.

QTY - USDT

수량 모드 - USDT 환산 수량 모드 전환 버튼.

6) 리스크·보호 옵션

Auto Stop Market Percent (%)

기본값: 0.5. SETTING에서 지정한 퍼센트 값에 따라 오토 스탑 (AT SL 버튼) 기능 활성 시, 해당 비율 기준으로 스탑 마켓 주문이 자동 생성됩니다.

Price Protection

스탑마켓 주문 트리거 발동 시점에 현재 시장가격과 트리거 가격의 괴리율이 거래소 허용 범위를 초과하면 주문이 거부되도록 하는 안전 장치입니다.
- ON: 급격한 가격 변동 시 비정상 체결 방지 (변동성 장세·중·장기 주문에 유리)
- OFF: 괴리율과 무관하게 즉시 발동·체결 (초단타·스캘핑에 유리)
일반적으로 스캘핑·단타에서는 OFF를 권장합니다.

7) 주문/포지션 실행

  • Buy/Long, Sell/Short
  • Cancel Orders: 현재 심볼 미체결 전부 취소
  • Close Position: 현재 심볼 포지션 시장가 청산
  • Reduce-Only 체크 지원
  • Auto-Reduce: 포지션 종료 주문 시 Reduce-Only가 자동 활성화되어 역방향 신규 진입을 방지하는 옵션
  • TIF: GTC 고정
주문/체결/취소 사운드는 버튼 클릭이 아니라 실제 응답 이벤트를 기준으로 재생됩니다.

8) 잔고·코스트·설정

  • BALANCE: 계정 잔고 및 미실현 PnL
  • COST VALUE: 좌클릭시 밸류 업데이트, 우클릭시 예상 청산가 및 수수료 안내 팝업
  • Pin: 항상 위
  • Default: 설정 팝업 ( 테마 / 사운드 / 오토 스탑 % )

9) SFX·테마

  • SFX: 원하는 사운드 팩 선택. 동일한 양식으로 커스터마이징 자유롭게 가능.
  • THEME: 원하는 테마 선택. 동일한 양식으로 커스터마이징 자유롭게 가능

10) 단축키

고정 단축키

  • Alt: AMOUNT 절반(누르고 있는 동안)
  • Caps Lock: Reduce‑Only 토글
  • ESC: Cancel Orders 실행

사용자 정의

Ctrl+숫자 또는 F13~F22로 BBO 타입 전환 및 매수/매도/청산 등 매핑.

{
  "Ctrl+1": "Queue 1",
  "Ctrl+2": "Counterparty 5",
  "Ctrl+3": "Grid 0.5%",
  "F13": "Buy", "F14": "Sell", "F15": "Close Position",
  "F16": "Halve Amount", "F17": "Double Amount"
}

shortcut_settings.json 파일에서 정의·수정 후 프로그램 재실행.

11) 크롬 디버깅 모드 연동(자동 심볼 감지)

바탕화면 바로가기 대상 경로 예시:

"C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --user-data-dir="C:/ChromeDebugProfile"

디버깅 모드의 바이낸스 선물 탭 URL에서 현재 심볼을 자동 추적합니다.

동봉된 바로가기를 사용해도 됩니다.

1) 소개

Trades는 바이낸스 스트림의 실시간 체결내역 과 청산 주문 이벤트 뷰어 입니다.

본 프로그램은 개발자 모드 크롬 브라우저와 연동됩니다..

2) 메인 UI

가격/체결 테이블

  • 좌측 열: 가격
  • 우측 열: 체결 규모(USD) — price × qty
  • 컬러 코딩: BUY, SELL
  • 강조: 설정 임계값 이상 체결은 폰트 확대/굵게 표시

상단 정보·컨트롤

  • Title Bar: Cap : 시가총액 - 마지막 수신시각 형식
  • 심볼 라벨: 현재 스트림 심볼(예: BTCUSDT)
  • 순간 대금: 3초 누적 매수/매도 시장가 체결건 USDT 환산.

메인 버튼

  • PIN: 창을 항상 위에 고정(토글). 상태는 설정에 저장·복원됩니다.
  • LIQ: 청산 다이얼로그 열기/닫기 (자세한 설명은 아래 참조)
  • PERP: 선물/현물 모드 전환(현물 지원 심볼일 때만 Spot 활성)
  • SET: 설정 다이얼로그 열기

3) 청산(Force Order) 다이얼로그

바이낸스 forceOrder 이벤트를 실시간 표로 표시합니다. 현재 UI 심볼과 일치하는 청산만 기록하여 과거 잔여 콜백을 차단합니다.

표시 항목

  • TIME / SIDE / QTY / PRICE / VALUE
  • 가치(Value)는 K/M/B 단위 축약 표기
  • 최대 100건 유지, 최소값 필터로 실시간 정제

옵션

  • AUTO POPUP (3s): 신규 청산 발생 시 자동 팝업(3초 후 자동 숨김)
  • ALERT SOUND: 청산 알림음 재생(중복 재생 방지)
  • MIN SIZE: 이 값 이상인 청산만 표/알림에 반영

NEW 배지 & 읽지 않음 표시

오토팝업을 끄고 창이 닫혀 있는 상태에서 조건 충족 청산이 발생하면, 메인 UI의 LIQ 버튼에 NEW 깜빡임이 표시됩니다.

통계 라벨

  • 1M: 최근 1분 청산 LONG/SHORT 누적
  • 3S: 최근 3초 청산 LONG/SHORT 누적

4) 현물/선물 전환 (PERP/Spot)

  • PERP(Futures, 선물) 버튼 클릭 시, 심볼이 현물로 지원되면 Spot으로 전환/복귀합니다.
  • 현물 미지원 심볼은 버튼이 비활성화되며, 자동으로 PERP로 유지됩니다.

5) 설정 다이얼로그

Rows

표시 행 수(10~100). 적용 시 즉시 테이블 레이아웃에 반영됩니다.

Highlight Large Trades

대량 체결내역 강조 온/오프와 임계값(USD) 지정.

PIN on Top

항상 위 상태 저장·복원. 메인 PIN 토글과 동기화됩니다.

Apply / Reset / Cancel

  • Apply: 저장 및 재연결(스트림/UI 리프레시)
  • Reset: 기본값으로 초기화(시가총액 초기화 포함)
  • Cancel: 변경 취소

6) 단축키

  • ESC: 기본 종료(운영체제/환경에 따라 다를 수 있음)
  • 일부 환경에서는 심볼 전환·포커스 이동 등 사용자 정의 단축키를 보조 툴로 매핑해 사용할 수 있습니다.

7) 문제 해결 / 팁

  • 데이터 지연/미수신: 네트워크 상태, 바이낸스 상태 점검. 필요 시 프로그램 재시작.
  • 현물 전환 불가: 해당 심볼이 현물 미지원일 수 있습니다. PERP 유지.
  • 청산 알림음 미재생: liq.wav 파일 존재 여부와 오디오 장치를 확인하세요.
  • 로그 확인: 오류·경고는 log_trades.log를 확인하세요.
