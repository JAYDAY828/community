서비스 이용약관

부평구금융인 Binance API 서비스 이용에 관한 규정입니다.

제1조 (목적)

이 약관은 부평구금융인 BINANCE API의 이용과 관련하여, 이용자와 운영자 간의 권리·의무 및 책임사항을 규정함을 목적으로 합니다.

제2조 (운영 성격 및 후원)

  • 부평구금융인 BINANCE API는 후원 기반으로 운영됩니다.
  • 이용자는 후원 금액에 따라 일정 기간 이용 혜택을 받을 수 있으며, 기간은 운영 상황에 따라 변동될 수 있습니다.
  • 이용 제공 여부 및 기간은 후원 상황, 기술적 여건, 기타 운영 사정에 따라 조정될 수 있습니다.

제3조 (개인정보 및 접속 정보 수집)

부평구금융인 BINANCE API는 부정 사용 방지 및 안정적인 이용 제공을 위해 다음 정보를 수집·저장할 수 있습니다.

  • 접속 시간
  • IP 주소
  • 기기 정보 (Machine ID)
  • 시리얼키 사용 기록

수집된 정보는 부정 접속 적발 및 운영 목적 외에는 사용하지 않으며, 이용자가 탈퇴하는 시점까지 보관한 후 지체 없이 파기합니다.
다만, 관련 법령에 따라 일정 기간 보존이 요구되는 경우에는 해당 법령에서 정한 기간 동안 별도로 보관할 수 있습니다.
부정 사용이 적발된 경우, 이용자의 계정은 사전 경고 없이 영구 정지될 수 있습니다.

* 부평구금융인 BINANCE API를 이용하는 경우, 개인정보 수집에 동의한 것으로 간주됩니다.

제4조 (책임 제한)

면책 조항 (Disclaimer)

  • ● 본 도구는 바이낸스(Binance) API를 기반으로 작동하며, 모든 거래·체결·시세 데이터는 바이낸스 시스템에서 처리됩니다.
  • ● 부평구금융인 BINANCE API는 이용자의 지갑이나 자산에 직접 접근할 권한이 없으며, 이용자가 API 키 발급을 통해 허용한 기능 범위 내에서만 작동합니다.
  • ● 운영자는 바이낸스의 서버 장애, API 변경, 지연, 오류 등으로 인한 거래 문제에 대해 어떠한 책임도 지지 않습니다.
  • ● 이용자의 매매 손익, 주문 오류, 네트워크 장애, 계정 설정 실수 등으로 발생한 손실에 대해 운영자는 책임을 지지 않습니다.
  • ● 부평구금융인 BINANCE API는 투자 자문이나 재정적 권고를 제공하지 않으며, 모든 투자 판단과 책임은 이용자 본인에게 있습니다.

제5조 (이용 제한)

이용자가 다음 각 호에 해당하는 경우 이용을 제한하거나 종료할 수 있습니다.

  • 타인의 계정 정보 도용
  • 시리얼키 불법 공유 또는 판매
  • 부평구금융인 BINANCE API의 역분석·복제·변조 행위
  • 법령 및 본 약관 위반

위 사유로 인해 발생한 불이익에 대해 운영자는 책임을 지지 않습니다.

제6조 (지식재산권)

본 서비스는 바이낸스 API를 기반으로 제작되었으며, API 및 이를 통해 제공되는 모든 원천 데이터의 권리는 원저작자인 Binance (이하 “바이낸스”)에 귀속됩니다.

단, 본 서비스를 구성하는 소프트웨어, 디자인, 로고, 사용자 인터페이스(UI), 그리고 데이터를 가공·분석하여 생성된 2차적 산출물에 대한 저작권 및 지식재산권은 운영자에게 귀속됩니다.

이용자는 운영자의 사전 서면 동의 없이 다음 각 호의 행위를 하여서는 아니 됩니다.

  • ● 본 서비스의 소스 코드 또는 프로그램을 역설계(Reverse Engineering), 디컴파일(Decompile), 복제하는 행위
  • ● 본 서비스를 수정, 임대, 배포, 판매하거나 이를 기반으로 2차적 저작물을 작성·제공하는 행위
  • ● 서비스에 포함된 운영자의 상표, 로고, 고유한 식별표지를 무단으로 사용하는 행위

제7조 (기타)

  • 본 약관에서 정하지 않은 사항은 관계 법령 및 상관례를 따릅니다.
  • 운영자는 필요한 경우 본 약관을 변경할 수 있으며, 변경된 약관은 공지 또는 안내 후 적용됩니다.
Q&A

자주 묻는 질문

서비스 이용 중 자주 발생하는 질문들을 모았습니다.

시스템

프로그램이 실행되지 않거나 바로 꺼집니다.

윈도우 11의 스마트 앱 컨트롤(SAC) 기능이 켜져 있을 경우, 인증서가 없는 개인 개발 프로그램의 실행을 원천 차단할 수 있습니다.

1. 윈도우 [설정] → [앱 및 브라우저 컨트롤]을 검색하여 이동합니다.
2. [스마트 앱 컨트롤 설정]을 클릭합니다.
3. 설정을 [끄기]로 변경 후 프로그램을 다시 실행해주세요.

* Windows Defender(바이러스 백신)는 끄지 않으셔도 정상 작동합니다. SAC만 해제해주세요.
설정

API Key를 입력했는데 작동하지 않아요.

다음 체크리스트를 확인해주세요:

  • API 생성 시 Enable Futures에 체크했는지 확인
  • IP 제한 설정에 현재 사용 중인 IP가 올바르게 등록되었는지 확인
  • 복사 붙여넣기 과정에서 공백(스페이스바)이 들어가지 않았는지 확인

위 사항을 확인 후에도 안 된다면 기존 키를 삭제하고 새로 발급받는 것을 권장합니다.

결제/후원

후원 기간이 끝나면 어떻게 되나요?

프로그램 로그인 시 메인 화면에 만료일(Expiration Date)이 항상 표시됩니다.
만료일 자정이 지나면 로그인이 즉시 제한되므로, 끊김 없는 사용을 원하시면 날짜를 확인하시고 미리 텔레그램이나 카카오톡으로 연장 문의를 부탁드립니다.

기능

모바일에서도 사용할 수 있나요?

본 툴킷은 윈도우 기반의 PC 전용 프로그램입니다.

단, 후원자분들께는 Crypto Detector와 동일한 로직의 트레이딩뷰(TradingView) 전용 지표를 제공해 드립니다.
트레이딩뷰 앱의 '얼러트(Alert)' 기능을 활용하시면, 모바일 환경에서도 동일한 시세 포착 알림을 실시간으로 받아보실 수 있습니다.

업데이트

자동 업데이트가 진행되지 않습니다.

자동 업데이트와 버전 체크는 전용 런처(Launcher.exe)를 통해 실행할 때만 작동합니다.
폴더 내의 본 실행파일을 직접 켜지 마시고, 반드시 Launcher 아이콘으로 실행해주셔야 최신 기능과 보안 패치가 적용됩니다.

Guide

Binance API Key 발급

안전한 툴킷 사용을 위한 API 키 발급 및 설정 가이드입니다.

1) 발급 절차

  1. 바이낸스 로그인 후 우측 상단 [Account] → [API Management] 메뉴로 이동합니다.
  2. [Create API] 클릭 후 System generated 방식을 선택합니다.
  3. Label(이름)을 입력합니다. (예: futures-toolkit)
  4. 이메일 및 OTP 인증(Security Verification)을 완료하여 키를 생성합니다.

2) API 권한 설정 (Restrictions)

반드시 체크 (허용)

  • Enable Reading
  • Enable Futures

체크 해제 (금지)

  • Enable Spot & Margin Trading
  • Enable Withdrawals (출금)

본 툴킷은 Enable Futures(선물 거래) 권한만 있으면 모든 기능이 정상 작동합니다.
운영자는 개인 자산(현물 거래, 출금 등)에 대한 권한을 전혀 요구하지 않으므로 안심하셔도 됩니다.

3) IP 제한 설정 (보안 필수)

  1. API 설정 화면 하단의 IP Access Restrictions 항목을 확인합니다.
  2. Restrict access to trusted IPs only(권장) 옵션을 선택합니다.
  3. 네이버 검색창에 내 IP를 검색하여 본인의 IP 주소를 확인합니다.
  4. 확인된 IP 주소를 입력란에 붙여넣고 Confirm을 누른 뒤, 상단의 Save 버튼을 눌러 저장합니다.
허용된 IP 이외의 환경(다른 PC, 외부 네트워크 등)에서는 API가 동작하지 않아 안전합니다.

4) Order Toolkit에 입력

  1. 발급 완료 화면에 표시된 API Key와 Secret Key를 복사합니다.
  2. Order Toolkit 프로그램을 실행합니다.
  3. 상단 메뉴의 [API] 버튼을 클릭하여 복사한 키를 입력하고 저장합니다.

주의사항

  • ● Secret Key는 생성 직후 1회만 표시되며, 페이지를 벗어나면 다시 확인할 수 없습니다.
  • ● Secret Key를 분실한 경우 API를 삭제하고 새로 생성해야 합니다.

Crypto Detector

실시간 변동성 감지 및 시세 포착 툴킷 가이드입니다.

1) 소개 및 로그인

Crypto Detector는 조건에 맞는 필터로 종목을 빠르게 감지하고, 변동률·거래대금·미체결약정(OI) 정보를 기반으로 즉시 모니터링할 수 있는 툴킷입니다. 트레이딩뷰 지표 및 알림(Alerts) 기능과 연동하여 모바일 환경에서도 신호를 받아볼 수 있습니다.

모바일 알림 지원 (TradingView 연동)

본 프로그램은 PC 전용이지만, 외부 이동 시에도 시세를 놓치지 않도록 동일 로직의 트레이딩뷰(TradingView) 지표를 함께 제공합니다.

  • PC를 켜두지 않아도 24시간 서버 알림이 작동합니다.
  • 트레이딩뷰 앱의 '얼러트(Alert)' 기능을 통해 스마트폰 푸시 알림을 받을 수 있습니다.
TradingView : 지표 소개

보안 및 로그인 정책

  • ● 발급된 시리얼 키를 입력하여 로그인합니다. (무단 공유 적발 시 즉시 비활성화)
  • ● 로그인 연속 실패 시 보안을 위해 IP가 차단될 수 있으며, 이 경우 운영자에게 문의해야 합니다.

2) 감지 로직 & 필터

메인 감지 기준 (Main)

  • AND 조건 아래 두 조건을 모두 충족 시 감지
  • CHANGE %: 1분 등락률이 설정값 이상
  • 24H VOLUME: 24시간 거래대금 충족

신규 상장 (New Pair)

  • 예외 처리 신규 상장 직후 거래대금 부족 보완
  • ● 설정한 시간 동안 24H VOLUME 조건을 무시합니다.
  • ● CHANGE % 만 충족해도 즉시 감지합니다.

대형주 감지 (BTC/ETH/MAJ)

  • OR 조건 메인 필터보다 낮은 변동폭도 포착
  • ● 알트코인 기준 설정 시 대형주 감지 누락을 방지합니다.
  • ● 별도의 감지 세트(낮은 %와 높은 거래대금)가 적용됩니다.

빅 캔들 (Big Long/Short)

  • 보조 감지 거래대금 미충족 종목의 급등락 포착 (상장빔 등)
  • ● 24H VOLUME 미충족 종목에서 강한 캔들 발생 시 감지합니다.
  • ● 설정값(10%~30%) 이상의 1분 변동 발생 시 알림을 보냅니다.

서킷 브레이커 (Circuit Breaker)

시장 전체가 급락하여 하락 방향 BIG CANDLE이 동시다발적으로 폭증할 경우, API 과부하 및 렉 방지를 위해 BIG CANDLE 기능만 일시적으로 자동 비활성화됩니다.

* 메인 감지 및 대형주 감지 기능은 정상 작동합니다.

4) 문제 해결 및 팁

  • 리스트가 비어 있습니다. 필터 설정값(Change %, Volume)이 현재 장세에 비해 너무 높을 수 있습니다. 설정값을 조금 낮춰보세요.
  • 클립보드 복사가 안 됩니다. SET 창에서 CLIPBOARD SYNC 옵션이 체크되어 있는지 확인하세요.
  • 알림 소리가 나지 않습니다. 설정에서 Sound가 ON인지 확인하고, 프로그램 폴더 내 SFX 폴더에 사운드 파일이 있는지 점검하세요.

Order Toolkit

바이낸스 USDⓂ 선물을 위한 고성능 주문 실행 툴킷입니다.

1) 소개 및 주요 특징

Order Toolkit은 바이낸스 정식 API를 기반으로 안전한 자산 보호와 빠른 주문 속도를 제공합니다. 기본적으로 크롬 브라우저와 연동되며, 필요 시 Manual Mode로 단독 운용할 수 있습니다.

그리드 자동호가

고빈도 봇으로 인한 호가 밀림(슬리피지) 현상에 대응합니다. 현재가 주변에 다중 지정가 주문을 살포하여 체결 확률을 높이고 수수료(Maker)를 절약합니다.

Auto Stop Market

돌파 진입이나 급변동 시 리스크 관리를 위해, 진입과 동시에 설정된 % 구간에 자동으로 스탑로스(시장가) 주문을 대기시킵니다.

2) 시리얼 키 인증

  • 발급받은 Serial Key를 입력하여 로그인합니다.
  • REMEMBER ME 체크 시 키가 암호화되어 저장됩니다.
  • 3회 인증 실패 시 프로그램이 종료되며, 부정 사용 적발 시 경고 없이 키가 삭제될 수 있습니다.

3) 크롬 연동 (Auto - Manual)

툴킷과 종목 연동을 위해, 크롬이 디버깅 모드로 실행되어야 심볼이 자동 동기화됩니다. (AUTO 버튼)

간편 실행 방법 (권장)

툴킷 설치 폴더 내에 있는 [Chrome Debug Mode] 바로가기 아이콘을 실행하면, 별도의 설정 없이 즉시 연동 모드로 브라우저가 열립니다.

수동 바로가기 생성 시 대상(Target) 경로 예시:
"C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --user-data-dir="C:/ChromeDebugProfile"
Tip: 툴킷 상단의 Auto 버튼을 클릭해도 크롬 디버깅 모드가 자동으로 실행됩니다.

4) 상단 메뉴 설정

  • LEVERAGE 1x ~ 125x 배율 선택 (현재 포지션 정보와 실시간 동기화)
  • MARGIN Cross(교차) / Isolated(격리) 모드 토글
  • SYMBOL 현재 종목 표시. 입력창 클릭 시 수동 입력(예: BTC) 가능하며 자동완성 지원
  • API 발급받은 API Key 리스트 관리 및 저장
* Add/Remove: 즐겨찾기 리스트 관리 / Auto/Manual: 크롬 자동 연동 여부 토글

5) 주문 UI 및 설정

주문 옵션 (Inputs)

  • ORDER (주문 타입) Limit (지정가), Market (시장가), Stop Market 지원
  • BBO (호가 모드)
    • Queue n: 내 호가(매수 시 매수대기) n번째 줄 배치
    • Counterparty n: 상대 호가(매수 시 매도대기) n번째 줄 즉시 체결
    • Grid x%: Queue 1 기준 현재가 ±범위에 분할 주문 (스캘핑 권장)
  • PRICE (가격) BBO 모드 시 자동 계산 (잠금). 수동 입력 가능.
    라벨 좌클릭: 매수1호가 / 우클릭: 매도1호가 즉시 지정

기능 토글 (Toggles)

  • AT SL
    Auto Stop Market
    활성화 시 포지션 진입과 동시에 스탑로스(SL) 주문을 자동으로 실행합니다.
  • BBO
    Best Bid / Offer
    최우선 호가 자동 추적. 설정된 모드(Queue/CP)에 따라 주문가를 실시간 보정합니다.
  • SYNC
    Price Sync
    클립보드, 트레이딩뷰, 호가창 툴킷의 가격을 주문 입력창에 즉시 동기화합니다.
  • QTY
    Unit Toggle
    QTY(코인 개수) ↔ USDT(금액 환산) 입력 단위를 전환합니다.

수량 및 금액 제어 (마우스 조작)

Quick Amount (1K ~ 1M)
좌클릭 : 수량 추가 (+)
우클릭 : 수량 감소 (-)
Cost Value (예상 소요 금액)
좌클릭 : 금액 갱신 (현재가)
우클릭 : 청산가/수수료 안내

6) 리스크 관리 및 실행

Auto Stop Market (%)

AT SL 기능 사용 시 적용될 퍼센트입니다.

기본값: 0.5% (설정 변경 가능)

Price Protection

  • ON: 괴리율 초과 시 주문 거부 (안전) 변동성 장세
  • OFF: 괴리율 무관 즉시 발동 (속도) 스캘핑 권장

주문 실행 및 포지션 관리

Buy/Long Sell/Short Cancel Orders (ESC) Close Position
Reduce-Only: 포지션 감소만 허용
Auto-Reduce: 포지션 종료 주문 시 Reduce-Only 자동 적용

7) 잔고 및 기타 설정

  • BALANCE: 계정 잔고 및 미실현 손익(PnL) 표시. 클릭 시 표시 항목 전환.
  • PIN: 창을 항상 위에 고정
  • SETTING (Default): 테마(Theme), 사운드(SFX), 오토 스탑 비율 등 전반적인 설정 팝업
  • SFX/THEME: 사운드 팩 및 테마 스킨을 커스터마이징하거나 선택할 수 있습니다.

8) 단축키 (Shortcuts)

고정 단축키

  • Alt (Hold): 수량(AMOUNT) 절반으로 임시 변경
  • Caps Lock: Reduce-Only 토글
  • ESC: Cancel Orders (미체결 취소)

사용자 정의 (shortcuts.json)

shortcut_settings.json 파일에서 키 매핑 수정 가능

{
  "Ctrl+1": "Queue 1",
  "F13": "Buy", 
  "F14": "Sell", 
  "F15": "Close Position"
}

9) 문제 해결

  • 실행 불가: 전체 삭제 후 런처를 재실행하거나 관리자 권한으로 실행해보세요.
  • 주문 실패: 거래소 과부하 또는 API 권한/IP 제한 설정을 확인하세요.
  • 연동 불가: 크롬 디버깅 포트(9222)가 다른 프로그램과 충돌하는지 확인하거나 프로필 경로를 점검하세요.
  • 시간 동기화: PC 시간이 서버 시간과 오차가 크면 API 요청이 거부됩니다. 윈도우 [날짜 및 시간 설정]에서 '지금 동기화'를 눌러 시간을 맞춰주세요.
면책: 프로그램 사용 중 발생한 손익은 전적으로 사용자 책임입니다. API Key 및 Secret Key는 타인에게 노출되지 않도록 각별히 주의하세요.

Real-time Order Book

웹보다 빠른 실시간 호가창 및 매물대 분석 뷰어입니다.

1) 소개 및 주요 특징

바이낸스 웹사이트의 호가창(Snapshot 방식)은 갱신 주기가 느려 스캘핑에 불리합니다. 본 뷰어는 **Socket 기반 리얼타임 호가창**으로, 결정적인 순간의 매수/매도 벽과 체결 강도를 실시간으로 포착합니다.

실시간 USDT 환산

모든 호가 물량을 코인 개수가 아닌 USDT(금액) 가치로 자동 환산하여 표시합니다. 허수 주문과 실제 매물대를 직관적으로 파악할 수 있습니다.

3초 누적 체결량

순간적인 매수/매도세의 힘을 시각화하기 위해, 최근 3초간 체결된 시장가 주문의 총액을 중앙에 표시합니다.

강력한 모아보기

틱(Tick) 단위뿐만 아니라 10, 100, 1000 단위의 Aggregation(모아보기)을 지원하여 시장의 깊이(Depth)를 다각도로 분석합니다.

툴킷 연동

호가창의 특정 가격을 클릭하면 Order Toolkit의 가격 입력창에 즉시 반영되어 빠른 지정가 주문이 가능합니다.

2) 메인 UI 및 컨트롤

상단 컨트롤 패널

  • GLOBAL / SYMBOL 모아보기(Aggregation) 설정을 모든 종목에 공통 적용할지(GLOBAL), 각 종목별로 개별 저장할지(SYMBOL) 선택합니다.
  • Aggregation 호가를 묶어서 볼 단위 (1, 10, 100, 1000) 선택. 숫자가 클수록 넓은 범위의 매물대를 보여줍니다.
  • FUTURES / SPOT 선물(Futures)과 현물(Spot) 마켓 데이터를 전환합니다. (현물 상장 종목일 경우에만 활성화)

호가창 테이블 상세

  • 매도(Ask): 상단에 위치, 붉은색 계열
  • 매수(Bid): 하단에 위치, 초록색 계열
  • 대량 주문(●): 설정된 기준을 초과하는 '왕건이' 물량 강조
중앙 체결가 영역
  • • 마지막 체결 가격 및 등락 방향(↑/↓)
  • • 좌측: 최근 3초간 누적된 매수 체결액
  • • 우측: 최근 3초간 누적된 매도 체결액

3) 크롬 연동 (심볼 동기화)

Order Toolkit과 동일한 방식을 사용하여, 웹 브라우저의 종목 변경을 감지하고 호가창을 자동으로 전환합니다.

필수: 크롬 디버깅 모드 실행 (바로가기 활용)
"C:\...\chrome.exe" --remote-debugging-port=9222 --user-data-dir="C:/ChromeDebugProfile"

4) 문제 해결

  • 데이터가 표시되지 않음: 네트워크 상태를 확인하거나 프로그램을 재시작하세요. 바이낸스 소켓 연결이 일시적으로 끊겼을 수 있습니다.
  • 크롬 연동 불가: 크롬이 디버깅 모드(포트 9222)로 실행되었는지 확인하세요. 일반 모드의 크롬 창이 켜져 있다면 모두 종료 후 재실행해야 합니다.
  • 프로그램 지연: 거래량이 폭발하는 시점에는 데이터 처리량이 급증하여 일시적인 지연이 발생할 수 있습니다.
  • 오류 로그: 프로그램 폴더 내 log_orderbook.log 파일을 확인하면 원인 파악에 도움이 됩니다.
Tape Reading

Trades & Liquidation

실시간 체결 내역 필터링 및 청산(Force Order) 감지 뷰어입니다.

1) 소개 및 주요 특징

바이낸스 웹에서 제공하는 체결내역은 단순 스냅샷으로, 시장의 속도와 중요 체결을 놓치기 쉽습니다. 본 툴은 **거래대금 필터**를 통해 '의미 있는 체결'만 선별하여 보여주며, **청산(Liquidation)** 이벤트를 실시간으로 포착합니다.

거래대금 필터링

설정된 금액(USD) 미만의 자잘한 체결은 걸러내고, 고래들의 대량 체결만 시각적으로 강조하여 흐름을 파악합니다.

청산(Liquidation) 감지

단기 추세의 변곡점이 될 수 있는 강제 청산(Force Order) 발생 시, 알림음과 팝업으로 즉시 알려줍니다.

2) 메인 UI 구성

체결 테이블 (Main Table)

  • Price 체결 가격
  • Value 체결 규모 (USD 환산 금액)
  • Color BUY / SELL
  • Visual 대량 체결 시 폰트 확대 및 굵게 강조

상단 정보바 (Top Bar)

  • Symbol Info: 현재 연결된 종목명 및 시가총액(Cap)
  • 3초 순간 대금: 최근 3초간 발생한 매수/매도 시장가 체결 총액을 표시하여 순간 매매 강도를 보여줍니다.

컨트롤 버튼

LIQ : 청산 다이얼로그 열기/닫기
PERP : 선물(Futures) / 현물(Spot) 전환
SET : 설정 메뉴 (필터링 금액 등)
PIN : 항상 위 고정

3) 청산(Liquidation) 감지

Force Order 이벤트를 별도의 팝업창에 기록합니다. 다른 툴킷으로 종목 변경 시, 해당 종목의 청산 내역만 남기고 이전 기록은 자동 삭제됩니다.

표시 데이터

  • TIME : 발생 시각
  • SIDE : 청산 방향 (Long / Short)
  • PRICE : 청산 가격
  • QTY / VALUE : 청산 물량 및 금액 (K/M/B 단위 축약)
  • * 1M / 3S 통계를 통해 1분/3초간 누적된 청산 규모를 보여줍니다.

알림 옵션 (Alerts)

  • AUTO POPUP (3s) 청산 발생 시 자동 팝업 후 3초 뒤 숨김
  • ALERT SOUND 청산 발생 시 경고음 재생
  • MIN SIZE 설정된 금액 이상의 청산만 알림
Tip: 오토 팝업을 끈 상태에서 창이 닫혀있더라도, 조건에 맞는 청산이 발생하면 메인 UI의 LIQ 버튼에 NEW 배지가 깜빡여 알려줍니다.

4) 설정 및 마켓 전환

설정 (SET)

  • Highlight Large Trades 설정한 금액(USD) 이상의 체결 건을 굵은 폰트로 강조하여 보여줍니다. (필터링 핵심)
  • Rows 테이블에 표시할 최대 행 개수를 조절합니다. (10 ~ 100)

마켓 전환 (PERP/Spot)

선물(Futures) 데이터뿐만 아니라 현물(Spot) 체결내역으로 전환하여 볼 수 있습니다.

* 단, 해당 심볼이 현물 마켓에 상장되어 있지 않은 경우 버튼이 비활성화됩니다.

5) 문제 해결

  • 데이터가 멈추거나 지연됩니다. 네트워크 상태를 확인하세요. 바이낸스 소켓 연결이 끊긴 경우 SET → Apply를 눌러 재연결을 시도하세요.
  • 현물(Spot) 전환이 안 됩니다. 해당 코인이 바이낸스 현물 마켓(USDT)에 상장되어 있는지 확인해주세요. 미지원 시 PERP 모드로 고정됩니다.
  • 청산 알림음이 안 들립니다. 프로그램 폴더 내 SFX 폴더에 liq.wav 파일이 존재하는지, 윈도우 볼륨 믹서 설정을 확인하세요.
